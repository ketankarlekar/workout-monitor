# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Workout Monitor, please **do not** open a public issue. Instead, please email the maintainer directly with details about the vulnerability.

### What to Include
- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fix (if you have one)

We take all security reports seriously and will work to resolve them as quickly as possible.

## Security Best Practices

### For Users
- Keep the app updated to the latest version
- Don't share your password with anyone
- Clear app cache and data if removing the app

### For Developers
- All code changes are reviewed before merging
- Dependencies are regularly updated
- Use TypeScript for type safety
- Follow OWASP guidelines for mobile security

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | ✅ Yes    |

## Dependency Security

We use tools to scan for known vulnerabilities in our dependencies:
- `npm audit` — Check for vulnerabilities
- Dependabot — Automated dependency updates

Dependencies are updated regularly to patch security issues.
